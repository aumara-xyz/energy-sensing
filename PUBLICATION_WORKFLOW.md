# Publication Workflow

This repository is the public priority package for the Energy Sensing program.

The broader local working materials may remain private unless Peter explicitly requests a specific file be made public.

## Sync Rule

When local Energy Sensing files are updated on Peter's computer, the public package should also be checked.

If the update is public-facing, this repo should receive a distilled or direct public version of the change.

If the update is private, internal, raw, strategic, or clinically risky to publish casually, it should remain local.

## Public Package Rule

Public updates should preserve:

- falsifiability,
- medical non-claim discipline,
- GHP provenance limits,
- and the difference between a public research package and private working notes.

## Never Publish By Default

Do not publish by default:

- raw chats,
- credentials,
- private strategy,
- internal product/business notes,
- or unpublished sensitive working files.

## Public Release Discipline

When public-facing files change:

1. update local public package files,
2. commit the public package,
3. push to GitHub,
4. tag or release meaningful checkpoints,
5. report what changed and what stayed private.
